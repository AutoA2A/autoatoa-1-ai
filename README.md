# AI 教学互动小工具研究报告

## 课题与目标

本课题旨在开发一款轻量化的 AI 教学互动小工具，核心目标是生成随堂答题页面、知识点卡片、趣味课堂小游戏，并实现自动统计答题正确率、实时生成错题解析。工具需满足“无需额外后端服务”的约束，支持直接上传至 GitHub 网页托管平台生成分享链接，学生点击即可使用。

## 研究过程

综合各研究员思路后，确定以下技术方案：

- **核心功能**：随堂答题（单选/多选/判断）、知识点卡片（翻转展示）、趣味小游戏（填空/匹配/抢答）、正确率统计与错题解析。
- **关键技术**：采用单文件 HTML，内嵌所有 CSS 与 JavaScript 代码；使用 `localStorage` 实现答题数据本地持久化；通过 URL 参数传递题目数据或预设 JSON 题库；响应式布局适配手机与电脑。
- **用户流程**：教师通过页面内置编辑器定制题库或直接使用预置题库，生成分享链接；学生访问链接后，选择答题/卡片/游戏模式，完成互动后系统自动评分、记录错题并展示实时解析。所有数据本地处理，无后端依赖。

## 成品说明

成品文件为 `app.html`，主要功能如下：

1. **随堂答题**：支持单选题、多选题、判断题，答题即时反馈正误，完成后显示正确率与错题列表。
2. **知识点卡片**：点击翻转查看答案，支持展开/收起，便于课后复习。
3. **趣味小游戏**：随机抽题进行匹配或填空挑战，增加课堂互动性。
4. **自动统计**：使用 `localStorage` 记录答题历史，生成错题解析，帮助学生针对性巩固。
5. **分享便捷**：单文件上传 GitHub Pages 后生成链接，学生可直接访问，无需安装任何软件。

## 如何在 GitHub 上部署与运行

1. **创建仓库**：在 GitHub 新建一个公开仓库（如 `ai-teaching-tool`）。
2. **上传文件**：将 `app.html` 文件上传至仓库根目录。
3. **启用 GitHub Pages**：进入仓库 **Settings → Pages**，在 Source 中选择 `main` 分支和 `/ (root)` 文件夹，点击 Save。
4. **获取链接**：等待部署完成后，访问 `https://你的用户名.github.io/ai-teaching-tool/` 即可使用。
5. **分享**：将链接分享给学生，他们可直接在浏览器中打开并开始答题。
